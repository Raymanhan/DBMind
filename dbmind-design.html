02 — AI 核心模块

@table 语法与 AI 对话

这是 DBMind 与传统数据库工具的核心差异。用户通过自然语言描述需求，用 @table 精确告知模型使用哪些表，系统自动注入 Schema 上下文并生成可执行 SQL。

01
用户输入自然语言 + @table
在 AI 对话框中键入 @ 触发自动补全，选择目标表后描述查询意图。支持多表引用。
02
解析 @table 语法
系统提取所有 @table 标记，构建本次查询的表名列表。
03
拉取并注入 Schema
从元数据缓存中获取指定表的字段名、类型、外键关系，结构化注入 LLM 的 System Prompt。
04
LLM 生成 SQL
支持 OpenAI / Anthropic / Azure OpenAI / Ollama 本地模型，可配置切换。
05
SQL 校验 + 安全检查
语法校验、防 SQL 注入、危险操作（DROP/DELETE 无 WHERE）二次确认，生产库默认追加 LIMIT 1000。
06
注入 SQL 编辑器
生成的 SQL 带 AI 标记注入编辑器，用户可直接执行或继续编辑。支持多轮迭代优化。
// AI 对话面板
查询 @orders @order_items 最近 7 天每天的销售额汇总，按日期升序
AI 助手 · 已注入 Schema: orders(12列) order_items(8列)
已为您生成 SQL，通过 JOIN 关联两表统计每日销售额：
SELECT
  DATE_FORMAT(o.created_at, '%Y-%m-%d') AS date,
  SUM(oi.quantity * oi.unit_price) AS revenue
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
WHERE o.created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY date ORDER BY date ASC;
✦ AI 生成 · 点击编辑器可直接修改
// @TABLE 语法安全规则
✓ 白名单模式：可限制 AI 仅访问指定表
✓ 生产库自动追加 LIMIT 1000
⚠ DROP / DELETE 无 WHERE 需二次确认
✓ Schema 仅在本次对话内传递，不持久化到模型

04 — 技术栈

选型决策

每项技术的选择都有明确理由。Electron 保证跨平台，Vite 保证开发体验，可插拔 LLM 适配器保证灵活性。

e
桌面框架
Electron
一套代码覆盖 Windows + macOS。Node.js 生态原生访问系统 API，打包输出 .exe / .dmg。
R
前端框架
React + TypeScript
组件化架构，类型安全。配合 Vite 实现毫秒级热更新，提升开发体验。
CM
SQL 编辑器
CodeMirror 6
支持 SQL 语法高亮、智能补全、多种数据库方言，可扩展 @table 自定义语法。
AI
LLM 调用
可插拔 LLM 适配器
统一接口，支持 OpenAI / Anthropic / Azure OpenAI / Ollama 本地模型，用户自由选择。
Fa
后端服务
Node.js + Fastify
轻量高性能 HTTP 框架，处理数据库连接池、API 转发、查询缓存。
Ec
数据可视化
ECharts
结果集图表化展示。折线图、柱状图、散点图等，一键将查询结果转为可视化。
SQ
本地存储
SQLite（内嵌）
保存连接配置（加密）、查询历史、AI 对话记录。无需额外数据库服务。
eb
状态管理
Zustand
轻量 React 状态管理，替代 Redux。管理连接状态、查询结果、AI 对话上下文。
pk
打包发行
electron-builder
输出 NSIS .exe（Windows）和 .dmg（macOS）。支持代码签名与 macOS Notarization。

07 — 开发环境

Mac Intel 环境搭建

按顺序执行，总耗时约 1 小时。每步验证通过后再继续，避免依赖问题叠加。

1 · Xcode CLI Tools
~10 分钟
# Mac 底层编译依赖
xcode-select --install
# 验证
git --version
2 · Homebrew
~5 分钟
# macOS 包管理器
/bin/bash -c "$(curl -fsSL \
 https://brew.sh/install.sh)"
brew --version
3 · Node.js via nvm
~5 分钟
brew install nvm
nvm install 20
nvm use 20
node --version # v20.x
4 · VS Code + 插件
~10 分钟
brew install --cask visual-studio-code
# 必装：ESLint, Prettier, TypeScript
# SQLTools, ES7+ React Snippets
# GitLens, Thunder Client
5 · 本地数据库
~10 分钟
brew install mysql postgresql@16
brew services start mysql
brew services start postgresql@16
6 · 创建项目
~5 分钟
npm create electron-vite@latest dbmind
# 选择 react-ts 模板
cd dbmind && npm install
npm run dev # 窗口弹出 ✓
⚠
Mac Intel 特别注意
Intel Mac 的 Homebrew 路径为 /usr/local/bin， 与 Apple Silicon (/opt/homebrew) 不同，无需特殊配置。 使用 better-sqlite3 等原生 Node 模块时， 需用 electron-rebuild 重新编译，遇到报错属正常情况。 项目根目录创建 .nvmrc 写入 20 可固定 Node 版本。
