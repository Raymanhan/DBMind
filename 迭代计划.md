# DBMind 迭代计划

## 目标

下一阶段聚焦把 DBMind 从 MVP 推进到可长期使用的 MySQL 优先版本：

- MySQL 功能做到数据库管理软件的核心完整度。
- AI 模块兼容 OpenAI 格式的模型服务。
- AI Provider、Base URL、模型、Key、参数等配置放入设置页管理。
- 保持当前三栏工作台体验：左侧对象浏览器，中间 SQL 控制台，右侧 AI 助手。

## 里程碑 1：MySQL 连接与配置完善

### 功能范围

- 新建、编辑、删除、复制 MySQL 连接。
- 支持常用连接字段：
  - 连接名称
  - Host
- 
- 
  - Port
  - Username
  - Password
  - Database
  - Charset
  - Timezone
  - SSL
  - 连接超时
  - 只读模式
- 支持连接测试，返回明确错误信息。
- 支持保存连接配置到本地。
- 密码和 API Key 后续应接入系统 Keychain 或 Electron 安全存储方案。

### 验收标准

- 可以保存多个 MySQL 连接。
- 启动应用后连接列表可恢复。
- 测试连接成功后可以读取数据库列表和表结构。
- 连接失败时能展示 MySQL 原始错误的友好说明。

## 里程碑 2：MySQL 对象浏览器

### 功能范围

- 展示 MySQL 数据库列表。
- 展示当前数据库下的：
  - Tables
  - Views
  - Procedures
  - Functions
  - Triggers
  - Events
- 表节点展开后展示字段列表。
- 字段展示：
  - 字段名
  - 类型
  - 是否可空
  - 主键
  - 索引
  - 默认值
  - 注释
- 支持刷新 Schema。
- 支持搜索对象。

### 验收标准

- 切换数据库后对象树正确刷新。
- 表结构变化后点击刷新能看到最新结构。
- 搜索表名、字段名能快速定位。

## 里程碑 3：MySQL 表结构查看与 DDL

### 功能范围

- 表详情页：
  - 字段列表
  - 索引列表
  - 外键列表
  - 表注释
  - 建表 DDL
- 支持复制 DDL。
- 支持生成常见 SQL：
  - SELECT
  - INSERT
  - UPDATE
  - DELETE
  - COUNT
  - SHOW CREATE TABLE
- 支持表右键菜单。

### 验收标准

- 任意表可查看 `SHOW CREATE TABLE`。
- 右键表名可以快速生成 SQL 到编辑器。
- AI 面板可以复用当前表 Schema。

## 里程碑 4：SQL 编辑器与查询执行

### 功能范围

- 引入 CodeMirror 6 或 Monaco Editor。
- 支持 MySQL SQL 高亮。
- 支持多标签 SQL 控制台。
- 支持执行：
  - 当前选中 SQL
  - 当前语句
  - 全部 SQL
- 支持取消正在执行的查询。
- 支持查询超时。
- 支持查询历史。
- 支持收藏 SQL。
- 支持格式化 SQL。

### 安全规则

- 默认限制 SELECT 最大返回行数。
- 危险操作二次确认：
  - DROP
  - TRUNCATE
  - ALTER
  - DELETE 无 WHERE
  - UPDATE 无 WHERE
- 只读连接禁止写操作。

### 验收标准

- 可以执行多条 SQL 并展示每条结果。
- 查询错误能展示错误码、错误消息、SQL 片段。
- 大结果集不会卡死界面。

## 里程碑 5：MySQL 数据浏览与编辑

### 功能范围

- 表数据浏览。
- 分页查询。
- 条件过滤。
- 排序。
- 列宽调整。
- 复制单元格、复制行、复制结果。
- 行内编辑。
- 新增行。
- 删除行。
- 提交/回滚变更。
- 支持 NULL 值编辑。

### 验收标准

- 可以像传统数据库管理软件一样浏览和编辑表数据。
- 编辑操作生成可审计 SQL。
- 主键表支持精确更新；无主键表给出风险提示。

## 里程碑 6：结果集导出

### 功能范围

- 导出 CSV。
- 导出 JSON。
- 导出 Excel。
- 支持选择导出范围：
  - 当前页
  - 全部结果
  - 选中行
- 支持复制为 Markdown 表格。

### 验收标准

- 中文、日期、数字导出格式正确。
- 大结果集导出有进度反馈。

## 里程碑 7：AI 设置页

### 功能范围

设置页新增 AI 配置区域：

- Provider 类型：
  - OpenAI
  - OpenAI Compatible
  - Azure OpenAI
  - Ollama
  - Custom
- Base URL。
- API Key。
- Model。
- Temperature。
- Max Output Tokens。
- 请求超时。
- 是否启用流式输出。
- 测试模型连接。
- 默认 SQL 方言。
- 是否允许 AI 生成写操作。
- 是否默认追加 LIMIT。

### OpenAI 兼容格式

优先兼容以下接口风格：

- `POST /v1/chat/completions`
- `POST /v1/responses`

推荐抽象统一接口：

```ts
interface AiProviderConfig {
  id: string;
  name: string;
  provider: 'openai' | 'openai-compatible' | 'azure-openai' | 'ollama' | 'custom';
  baseUrl: string;
  apiKey?: string;
  model: string;
  temperature?: number;
  maxOutputTokens?: number;
  timeoutMs?: number;
  streaming?: boolean;
}
```

### 验收标准

- 用户可以在设置页保存多个 AI 配置。
- 可以选择默认 AI 配置。
- 可以测试 AI 配置是否可用。
- 不再依赖命令行环境变量配置模型。

## 里程碑 8：AI 生成 SQL 能力增强

### 功能范围

- 支持 `@table` 自动补全。
- 支持多个表引用。
- 支持当前选中数据库的 Schema 注入。
- 支持表字段、索引、外键、注释注入。
- 支持多轮对话修改 SQL。
- 支持一键把生成 SQL 插入编辑器。
- 支持 SQL 解释。
- 支持 SQL 优化建议。
- 支持错误诊断。

### Prompt 规则

- AI 只能使用被 `@table` 引用的表，除非用户显式允许全库上下文。
- 默认只生成 SELECT。
- 写操作必须说明风险。
- 不允许编造不存在的字段。
- 输出结构化 JSON，包含：
  - SQL
  - 说明
  - 使用的表
  - 风险提示
  - 是否需要用户确认

### 验收标准

- 输入 `查询 @目标表 最近 7 天数据趋势` 可以生成可执行 MySQL SQL。
- 输入 `解释当前 SQL` 可以返回中文说明。
- 查询报错后点击 AI 诊断能解释原因并给出修复 SQL。

## 里程碑 9：本地存储与设置持久化

## 推荐开发顺序

1. 设置页与 AI 配置模型。
2. OpenAI Compatible Provider 抽象。
3. MySQL 连接配置持久化。
4. MySQL Schema 浏览器增强。
5. SQL 编辑器升级。
6. 查询执行、安全检查、结果集增强。

8. AI 多轮 SQL 生成、解释、诊断。
9. 导出、历史、收藏。
10. 打包发布。

## 技术注意事项

- MySQL 驱动继续使用 `mysql2/promise`。
- AI 请求必须从 Electron 主进程发起，避免 API Key 暴露到渲染进程。
- 渲染进程只通过 preload 暴露安全 IPC。
- 设置页保存时不要把敏感信息写入 React 状态日志或控制台。
- 生产环境默认禁止 AI 直接执行 SQL，只允许生成到编辑器，由用户确认执行。

## 成功标准

当以下路径全部顺畅时，可以认为 MySQL 优先版本完成：

1. 用户新建 MySQL 连接。
2. 用户在对象浏览器查看库、表、字段、索引、DDL。
3. 用户通过 SQL 控制台执行查询并查看结果。
4. 用户在设置页配置 OpenAI 兼容模型。
5. 用户输入 `@table` 自然语言需求。
6. AI 基于指定表结构生成 MySQL SQL 到编辑器。
7. 用户确认后执行 SQL，结果正确返回。
